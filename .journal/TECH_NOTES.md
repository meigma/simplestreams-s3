# Technical Notes

- V1 is governed by the [session 001 design](001/DESIGN.md) and its [companion plan](001/PLAN.md), and all five delivery phases are complete. The design prevails when the plan summarizes a requirement; design section 22 remains deferred.
- Use hexagonal architecture at all times. Keep business logic isolated from CLI, filesystem, network, storage, and other external adapters.
- Prefer functional testing before calling any feature complete. Unit tests are useful, but they do not prove the tool works the way the design intends.
- Take an agile approach to development. Avoid waterfall: underspecify when useful, prototype early, learn from the result, and refine from working behavior.
- Local development hosts have Lima and `mkcert` available. Prefer Lima for disposable Linux/Incus environments and `mkcert` for locally trusted HTTPS certificates when functional proofs need them.
- Phase 1 passed in closed, unmerged PR #7: Incus listed and imported the designed two-item VM catalog, and the imported fingerprint exactly matched the metadata-first combined SHA-256. Phase 2 preserved that wire contract and merged through PR #8 as `868a29c`.
- When `go-simplestreams` v0.1.0 enters the production module, select `golang.org/x/net` v0.57.0 or newer; Kusari flagged its transitive v0.52.0 for CVE-2026-39821. After installing a `mkcert` CA in a running Lima guest, restart Incus so it reloads the system certificate pool.
- Phase 3 merged through PR #9 as `d6cc2a7`: publication adopts compatible existing catalogs, verifies or repairs immutable content, and conditionally updates only `streams/v1/index.json` through bounded compare-and-swap retries. Phase 4 merged through PR #10 as `a505d04`: the proxy forwards validated ranges and conditions, bounds streams and shutdown, and caches readiness. Phase 5 merged through PR #12 as `45c019f`: the proxy exports the closed optional OTLP metric set, logs readiness transitions, and carries complete V1 operator/reference material plus required race and real-Incus CI acceptance.
- Phase 5 security closeout selected OpenTelemetry v1.44.0 and checksum-locked Go 1.26.5 after Kusari and `govulncheck` found newly published issues. Keep OTLP disabled when no endpoint is configured, restrict insecure export to loopback, preserve its fail-open behavior, and rerun current vulnerability analysis before releases.
- MinIO remains the routine build-tagged integration service through hidden endpoint, path-style, and single-PUT test hooks. It proves repeat and second-version publication, but it rejected transfer-manager multipart CRC-64/NVME completion with `InvalidPart`.
- A checkout-built `whzbox` at `9851e3b` successfully supplied and tore down disposable genuine AWS S3 environments after interactive login refresh. Phase 3 used it to prove create-only, matching-revision replacement, and stale-revision rejection; genuine-AWS multipart artifact behavior remains an optional extension.
- Keep integration infrastructure in scenario helpers but keep behavior and assertions in categorized process files; share one Testcontainers dependency across the full procedure to avoid resource churn.
- Repository settings and rulesets converge through `.github/scripts/configure_github_repo.py`; the Release Please GitHub App ID/private key are configured as repository variable/secret from 1Password and must never be committed.
- Operator documentation is the six-page Diátaxis set in `docs/docs/` (how-to: publish-images, deploy-the-proxy; reference: configuration, proxy-interface; explanation: design), live at https://meigma.github.io/simplestreams-s3/. Keep examples independently validated; the old operator-guide/observability/verification pages are gone.
- The repository is dual licensed Apache-2.0 OR MIT (`LICENSE-APACHE`, `LICENSE-MIT`, copyright 2026 Meigma). `melange.yaml` still lacks apk license metadata — pending follow-up.
- ghd tooling is removed; `.github/scripts/stage_release_assets.py` stages and integrity-checks GoReleaser assets into `dist/release-assets/`. The Release Dry Run workflow runs on any branch via `gh workflow run release-dry-run.yml --ref <branch>`.
- The repository's custom TypeScript Action is rooted at `action.yml`, with source, tests, dependencies, and the committed Node 24 bundle under `action/`. It installs `latest` or an explicit public CLI release after checksum verification, reuses `@actions/tool-cache`, invokes `publish` without a shell, and assumes callers already established AWS credentials.
- Evidence publication merged through PR #33 and shipped in v0.1.1. The Action accepts `attest-vm-image` v1.1.0's version-1 evidence manifest, requires a passing result and exact disk/metadata bindings, re-hashes the closed proof set, uploads content-addressed proofs before catalog activation, and advertises one custom `evidence-manifest` version item. Incus ignores that item and retains its exact list/import/fingerprint behavior; generic consumers must discover and verify the companion manifest themselves.
- Action and CLI versions share the root Release Please lane. A public stable `vX.Y.Z` release triggers the major-tag workflow to move `vX` to the exact release commit; exact release publication ignores moving-major tags. Release-PR Action CI uses the latest public CLI because the paired pending version does not exist until publication. Public v0.1.1 and moving tag `v0` both resolve to `d91dd69`.
- The v0.1.1 major-tag run passed but `actions/create-github-app-token` warned that `app-id` is deprecated; migrate the release workflows to `client-id` during maintenance.
- Default-setup CodeQL runs (`dynamic/github-code-scanning/codeql`) cannot be rerun via API; retrigger with an empty commit. After removing a worktree, golangci-lint's shared cache can emit phantom findings for the deleted path — run `golangci-lint cache clean`.
